# Runtime Security Lab

This lab track is the v1 deep exemplar for the Systems Security Engineering with Rust initiative.

- Learning goals: reason about Wasmtime execution limits, descriptor compatibility, and public-safe runtime failure modes.
- Intended lab type: synthetic WAT/Wasm guest modules plus runtime-policy validation.
- Required tooling: `cargo`, `cargo-nextest`, `mdbook`.
- Issue seed: `[Docs]: expand runtime-security labs into additional capability and telemetry exercises`.
